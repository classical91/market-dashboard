GOSH Smart Contracts ▼

GOSH represents a Git repository as a graph of smart contracts on the Everscale (TVM) blockchain. A single git push fans out into hundreds of parallel async contract deployments — each carrying a cryptographically verifiable piece of the commit history.

GoshRoot Top-level registry — maps usernames to wallets and DAOs. Entry point for all protocol interactions.
▼ deploys
GoshDao DAO contract governing a collection of repositories. Holds governance parameters, token supply, and voting logic.
GoshWallet Per-user identity contract. Signs Git operations with the user's key pair. Authorizes pushes and votes.
▼ contains
Repository One contract per repo. Stores branch heads and repository metadata.
Branch Stores the commit hash for a named branch tip (e.g. main).
Tag Lightweight and annotated Git tag objects stored on-chain.
▼ references
Commit Stores author, message, parent hash, and tree hash for each commit.
Tree Directory listing mapping filenames to blob/snapshot hashes.
Snapshot / Diff File content and deltas between commits. Large files chunked across multiple contracts.

Radicle Network ▼

Radicle's Heartwood protocol stores all collaboration data — issues, patches, comments — as signed Git objects within the repository itself, replicated across a gossip network of peer nodes. Your repo's identity is a rad:z… cryptographic ID that survives any server going offline.

🪪
Radicle ID (RID)
Each repository gets a permanent rad:z<base58> identifier derived from the delegate's public key. Stable and DNS-independent.
📡
Heartwood Protocol
TCP gossip protocol for repository replication. Seed nodes replicate repos they're interested in; protocol is fully open and self-hostable.
📝
COBs — Collaborative Objects
Issues, patches, and comments stored as signed Git refs inside the repo. Portable, offline-capable, and verifiable without a server.
🔑
Delegates & Patches
Delegates are the authorized keys for canonical refs. Patches are Radicle's pull-request equivalent — signed, stored as Git refs, and replicated across all peers.
🌐
Radicle Orgs (On-chain)
Optional Ethereum multisig/treasury contracts linking an on-chain org to a set of Radicle repos. Used for contributor rewards and DAO treasury management.
🛠
rad CLI
All peer operations — init, clone, push, patch, issue — are handled through the rad CLI. Standard Git remote rad:// protocol for compatibility.

Allo Protocol v2
Permissionless EVM smart contracts for grant allocation. Separates Pool creation, Strategy (funding logic), and token distribution. Strategies are pluggable modules.
📐
Quadratic Funding
Matching funds distributed proportional to the square root of contributor count — not dollar amount. Amplifies broad community support over whale donations.

Gitcoin Passport
Aggregates ENS, GitHub, Twitter, POAPs, and on-chain credentials into a humanness score. Gates Sybil-resistant participation in QF rounds via Verifiable Credentials.
🎯
Direct Grants
Fixed-allocation grants with milestone tracking, managed via Allo Protocol. Used by DAOs and foundations for structured grant programs without quadratic mechanics.

Protocol Comparison ▼

GOSH, Radicle, and GitHub represent three distinct points on the decentralization spectrum. This table compares key architectural properties relevant to on-chain developer infrastructure.

Property GOSH Blockchain Radicle P2P GitHub Centralized
Code storage On-chain smart contracts (Everscale) Git objects in gossip network Centralized servers (Microsoft)
Repository identity On-chain contract address Cryptographic RID (rad:z…) URL tied to domain/org
Commit history tamper-proof ✓ Blockchain-enforced ✓ Signed Git refs ✗ Mutable by admin
Censorship resistance ✓ Immutable contracts ✓ Multi-seed replication ✗ Platform can remove
Offline operation ✗ Requires blockchain node ✓ Full offline Git workflow ✗ Requires internet
Merge governance ✓ On-chain DAO votes ~ Delegate-signed patches ✗ Centralized maintainer
Issues & PRs ✓ DAO proposals / on-chain ✓ COBs (Git-native) ✓ Centralized platform
Token governance ✓ GOSH token ✓ RAD token (Ethereum) ✗ None
Blockchain dependency Everscale (required) Ethereum (optional, Orgs only) None
Standard Git client compat. ~ Via remote helper ✓ Via rad:// remote ✓ Native HTTPS / SSH
